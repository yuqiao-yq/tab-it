# Tab It 使用指南

> 一个替代浏览器新标签页的书签整理工具，支持 Chrome 与 Firefox。
> v0.20+ 在原有书签管理之上叠了一整套**可选**的 AI 能力：自动整理 / 打标签 /
> 语义搜索 / RAG 问答 / 自动备注 / 整理质检 / 相关阅读……所有 AI 功能都是
> **opt-in 默认关闭**，不配置任何 Provider 时与纯本地版本完全等价。
>
> 本文档同时是「顶栏 → 帮助」按钮里弹窗的内容来源，会随产品版本同步更新。

## 1. 初次使用

### 1.1 安装扩展

#### Chrome / Edge / 其他 Chromium 内核浏览器

1. 下载并解压 `tab-it-<version>-chrome.zip`（或自行执行 `pnpm build` 后用 `.output/chrome-mv3` 目录）。
2. 浏览器地址栏输入 `chrome://extensions`，打开右上角「**开发者模式**」开关。
3. 点击「**加载已解压的扩展程序**」，选择上一步的目录。
4. 安装完成后，**新开一个标签页**即可看到 Tab It。

> v0.15+ 起声明了 `host_permissions: <all_urls>` 用于**可选**的网页正文抓取功能（§9.3）。
> 浏览器扩展管理页会显示「读取所有网站的数据」。该权限**默认完全不使用**，
> 仅在你主动到「⚙ 设置 → 内容抓取」**勾同意 + 选范围**后才会发起任何 fetch；
> 抓到的内容**只存到本机 IndexedDB**，不上传任何服务器。

#### Firefox

1. 下载 `tab-it-<version>-firefox.zip`（或自行执行 `pnpm build:firefox`，输出在 `.output/firefox-mv2`）。
2. 浏览器地址栏输入 `about:debugging#/runtime/this-firefox`。
3. 点击「**临时载入附加组件**」，选择 zip 包内的 `manifest.json`（或 `firefox-mv2/manifest.json`）。
4. 临时安装的扩展会在浏览器关闭后失效，正式发布版可通过 AMO（addons.mozilla.org）签名后永久安装。

> **跨浏览器差异**：Tab It 在 Chrome 使用 MV3、Firefox 使用 MV2，并通过 `wxt/browser`
> 抹平 `chrome.*` API 的 Promise 差异，使用体验完全一致。

### 1.2 首次打开看到什么

- **顶栏**：左侧 Logo、中间网页搜索框、右侧「帮助」与「设置」两个按钮。
- **左侧分类栏**：默认空白，会提示「还没有分类」。
- **主内容区**：未选中分类时为空。
- **右下角 ✨ FAB**：默认**灰色**（未配置 AI）。点一下会引导你打开浮窗的 ⚙ 设置，
  添加任意 AI Provider（DeepSeek / OpenAI / 智谱 / Moonshot / Ollama 等 18 个预设可选）。
  **不配置也完全能用**：所有 AI 入口只是隐藏掉，非 AI 功能 100% 可用。

### 1.3 第一步建议：导入浏览器现有书签

打开「设置（⚙）」→「数据管理」→「**从浏览器导入书签**」，会按浏览器原有的文件夹层级把所有书签合并进来，保留分类结构。重复点击会做**幂等合并**：已存在的书签按 ID 更新，不会产生重复。

## 2. 顶栏

```
[ Tab It ]   [ 🔍 智能 / 网页 / 书签 / 标签 / AI 搜索框 ]   [ ? ]  [ ⚙ ]
```

### 2.1 统一搜索框（v0.10+ 支持 5 种模式）

由于扩展接管了新标签页，浏览器原生地址栏的「在新标签搜索」体验失效，
所以这里既能搜网页也能搜本地书签，并且通过前缀关键字切换模式：

| 输入 | 模式 | 行为 |
|---|---|---|
| 任意关键字 | **智能（默认）** | 实时本地搜索 + 主区切到搜索结果视图；回车若有匹配则打开首条，否则用所选引擎搜网页 |
| `@web 关键字` | **网页搜索** | 用所选引擎搜全网，本地结果不再触发 |
| `@bm 关键字` | **仅本地书签** | 强制本地搜索（语义化别名，回车不会跳网页） |
| `#标签名` | **标签筛选**（§5.3 / §8.5） | 按 tag 精确匹配显示所有命中卡片；顶部紫色横幅展示当前筛选 + ✕ 清除 |
| `@ai 关键字` | **AI 语义搜索**（§9.2） | 把 query embedding 化后按余弦相似度排序；需先生成 embedding |

**切换搜索引擎**：点击搜索框左侧的引擎图标（▾），可在 **Google / Bing / 百度 / DuckDuckGo** 四个之间切换；选择会持久化在 `localStorage`，下次进入沿用。

**反向同步**：当你点击卡片上的 tag chip 时，输入框会自动填入 `#xxx`，让你知道当前过滤状态。

### 2.2 帮助按钮

齿轮按钮**左侧**的问号图标即是帮助按钮，点击会打开本文档的弹窗。

- **关闭弹窗**：点击右上角 ✕、点击遮罩区域、或按 `Esc` 键。
- **滚动**：内容超过弹窗高度时，弹窗内部独立滚动，不影响背景页面。

### 2.3 设置按钮（齿轮 ⚙）

齿轮按钮是一个**气泡菜单**，点击展开三个二级入口：

| 入口 | 作用 |
|---|---|
| 数据管理 | 导入 / 导出 / 从浏览器同步 |
| 样式管理 | 主题、自定义壁纸、文字颜色 |
| 关于 Tab It | 版本号、UI build、GitHub |

> AI 设置**不在这里** —— 它在右下 ✨ FAB 打开的浮窗内的「⚙ 设置」Tab（§7.1）。
> 顶栏齿轮只管"书签数据 + 视觉样式"，避免两个齿轮概念混淆。

## 3. 数据管理

进入路径：**齿轮 → 数据管理**。弹窗内提供 3 个原子操作。

### 3.1 从浏览器导入书签

将当前浏览器（Chrome 或 Firefox 原生书签）一键合并到 Tab It：

- 按浏览器的**文件夹层级**还原为 Tab It 的「分类 / 子分类」结构。
- 已存在的书签按 ID **幂等合并**，不会产生重复。
- 适合**初次安装**或**新增了浏览器原生书签后**同步过来。

### 3.2 导入配置文件

选择一份 `tab-it-export-*.json` 文件（自己导出的 / 别人分享的）。导入前会弹出二次确认，让你选择「**合并**」还是「**替换**」：

| 模式 | 行为 | 适合场景 |
|---|---|---|
| 合并到现有数据（**推荐**） | 保留本地分类与书签，按 ID 合并；同 ID 取较新者，新数据追加到末尾 | 多设备同步、合并别人的分享 |
| 替换全部数据（**慎用**） | 清空本地所有分类与书签，并使用文件中的设置覆盖当前主题/布局 | 完全恢复到某次导出时的状态 |

> 文件需为 JSON 且包含 `categories` / `cards` 字段，否则会提示「未识别到分类或书签数据」。

### 3.3 导出配置文件

将当前所有分类、书签、设置打包为 JSON 文件下载到本地，文件名格式：

```
tab-it-export-YYYY-MM-DD.json
```

可用于备份、跨设备迁移、给同事分享自己的「精选书签集」等。

> **导出文件包含**：分类树、所有书签卡（含 tags / description / icon）、用户设置（主题、壁纸等）。
> **不包含**：最近使用记录、`localStorage` 的搜索引擎偏好、**AI 设置（Provider / apiKey）**、
> Embedding / 网页正文 / Tag 等本地索引（这些都在 IndexedDB 里，仅本机有效）。
> 这条约束是有意为之 —— **apiKey 等敏感信息绝不进入分享文件**。

## 4. 样式管理

进入路径：**齿轮 → 样式管理**。

### 4.1 主题

三种主题任意切换：

| 主题 | 行为 |
|---|---|
| ☀️ **明亮** | 强制亮色 |
| 🌙 **黑暗** | 强制暗色 |
| 🖥️ **跟随系统** | 默认。跟随操作系统外观偏好（`prefers-color-scheme`） |

### 4.2 自定义背景

#### 6 个预设壁纸

| 名称 | 风格 |
|---|---|
| 默认 | 回退到浅紫渐变（`global.css` 内置） |
| 极光 | 紫 → 蓝 → 青三段渐变 |
| 暮色 | 橙 → 粉双色渐变 |
| 深海 | 青 → 蓝双色渐变 |
| 林荫 | 绿色双段渐变 |
| 午夜 | 深蓝 → 深紫双段渐变 |

点击缩略图即时切换，已选项右下角会有 ✓ 标记。

#### 自定义渐变（调色盘）

预设之外，还内置了一个**轻量调色盘 + 多色渐变编辑器**，无需任何外部工具，几秒就能配出自己喜欢的壁纸。

| 控件 | 说明 |
|---|---|
| **类型切换** | `纯色 / 线性渐变 / 径向渐变` 三选一 |
| **色标列表** | 每行 = `调色盘按钮` + `Hex 文本框` + `✕ 删除`；首尾自动标注「起点 / 终点」，中间标注「中间 N」 |
| **+ 添加颜色** | 渐变模式下最多 **5** 个色标；纯色模式只保留 1 个 |
| **角度滑块** | 仅线性渐变可见。`0–360°` 滑块 + 数字输入框双向联动 |
| **实时预览** | 当前配置实时渲染成 `h-12` 矩形，所见即所得 |
| **CSS 文本** | 编辑器底部回显当前的 `background` 值，方便复制到博客 / 分享给同事 |
| **应用为壁纸** | 点击后才会写入设置（不实时同步），避免编辑过程中误触预设导致丢失 |

> **小贴士**
> - 调色盘用浏览器**原生 `<input type="color">`**，跨平台体验一致，无需安装额外字体或库。
> - Hex 文本框允许中间过渡态（例如正在敲 `#ab`），失焦或回车时校验，非法时会显红框。
> - 想保留这套渐变？复制 CSS 文本即可——所有渐变都用标准 `linear-gradient()` / `radial-gradient()`，可粘到任何 CSS 文件。

#### 自定义图片 URL

把任意图片链接粘到输入框（支持 `https://` 或 `data:image/...` 两种），按 **Enter** 或点「应用」即可。

#### 从本地上传

- 支持 PNG / JPG / WEBP / GIF 等浏览器认识的图片格式。
- **文件大小上限 2MB**。超出会拒绝并提示原因。
  - 原因：本地壁纸以 base64 存进 `chrome.storage.local`，该容器单条上限 ≈ 5MB，预留余量给其他数据。
- 上传后立即生效，且会持久化。

#### 清除背景

如果当前已设置了自定义壁纸，按钮区会出现红色「✕ 清除背景」按钮，一键回退到「默认」状态。

### 4.3 文字颜色（v0.7+）

样式管理弹窗末尾还有「文字颜色」section：6 个预设色块 + 调色盘 + Hex 文本框。
设置后会覆盖卡片标题等"未单独设色"的文字；按钮品牌色 / 辅助灰字不受影响。
设为空 → 回退到主题默认色。

## 5. 左侧分类栏

分类栏是 Tab It 的核心组织方式，支持**多层嵌套**、**拖拽排序**、**批量管理**等。

### 5.1 顶部按钮一览

```
[ ☰ 分类 ]                             [ ⇕ ] [ ☑ ] [ + ] [ ‹ ]
                                       展开/  批量  新建  收起
                                       折叠   管理  顶层  侧栏
                                       全部
```

| 按钮 | 行为 |
|---|---|
| **展开/折叠全部** | 一键展开 / 一键折叠所有有子分类的节点。当前没有任何子分类时禁用。 |
| **批量管理** | 进入多选模式（详见 5.5） |
| **+** | 新建一个**顶层**分类，会弹 prompt 输入名称 |
| **‹** | 收起整个侧栏到 40px 窄条 |

### 5.2 单条分类的交互

每行分类从左到右依次是：

1. **展开三角 ▶**：当且仅当该分类有子分类时显示，点击切换展开/折叠
2. **图标**：默认顶层 📁 / 子级 📂；点击进入「图标选择器」修改（详见 5.3）
3. **名称**：双击进入重命名（Enter 保存 / Esc 取消）
4. **数量**：递归统计该分类下（含所有子级）的书签总数
5. **+（hover 显示）**：在该分类下新建一个**子分类**，新建后自动展开父级
6. **🗑（hover 显示）**：删除该分类。删除前弹出确认；如果有子分类会提示「**及其所有子文件夹和书签**」

> **点击行为**：单击分类名称会进入该分类（主内容区刷新），同时如果有子分类会自动展开。

### 5.3 图标选择器（IconPicker）

点击任何分类前的图标，弹出 3-Tab 选择器：

- **Emoji**：从 64 个常用 emoji 中点选。
- **链接**：粘贴图片 URL，输入框下方实时预览，确认后点「应用」。
- **上传**：本地选图（**100KB 上限**，比壁纸更严格，因为图标会大量存储）。
- **重置**：一键恢复默认 emoji（顶层 📁 / 子级 📂）。

弹层会自动避让边界：靠右展开会自动右对齐；下方空间不足会自动向上展开。

### 5.4 添加子分类

两种方式：

1. **行内 +**：鼠标 hover 到目标父分类，点右侧出现的 +。
2. **拖拽嵌入**：把任意分类拖到目标分类**中部 40% 区域**（详见 5.6 拖拽规则），会成为目标的子节点。

### 5.5 批量管理

点击顶部「批量管理」按钮（☑ 图标）进入多选模式：

- 复选框只对**顶层分类**可用，子分类不能单独勾选（按整棵子树一起处理，避免遗留孤儿节点）。
- 顶部按钮变为：「✓ 全选 / ✕ 取消全选」、「🗑 批量删除」、「完成」。
- 删除前会提示：「确定删除 X 个分类（含子分类）、Y 个卡片吗？」
- 退出多选只需点「完成」。

### 5.6 拖拽分类（跨层级）

整行都是拖拽手柄。拖动有 **6px 距离阈值**，避免误触发（点击不会触发拖拽）。

拖到目标行时，根据**指针在目标行的纵向位置**决定语义：

| 区域 | 视觉指示 | 语义 |
|---|---|---|
| 上方 30% | 顶部出现一条 brand 色横线 | **before**：插入到目标**之前**（同层） |
| 中间 40% | 整行出现 brand 色环线 + 浅色背景 | **inside**：成为目标的**子节点**（自动展开父级） |
| 下方 30% | 底部出现一条 brand 色横线 | **after**：插入到目标**之后**（同层） |

**约束**：

- 不能把分类拖到自己的后代上（循环引用），系统会自动取消该次拖拽。
- 重命名 / 多选模式下禁用拖拽，避免冲突。

### 5.7 折叠 / 展开整个侧栏

点顶部 ‹ 按钮可以把整个侧栏收起到 40px 窄条，腾出空间给主内容。

收起态会：

- 显示展开 › 按钮。
- 显示**前 6 个**顶层分类的图标，点击单个图标会自动展开侧栏并跳转到该分类。

### 5.8 拖拽调整侧栏宽度

侧栏的右边缘有一个 **6px 拖拽热区**（hover 时会显示 2px brand 色细条）：

- **拖拽**：按住即可调整宽度，范围 **180 ~ 480px**。
- **双击**：恢复默认宽度（240px）。
- 折叠状态下隐藏热区（折叠后宽度固定 40px）。
- 宽度会持久化在 `tabit:settings` 中，下次打开沿用。

## 6. 主内容区

### 6.1 最近使用模块

只要在分类视图下，分类内容**上方**会常驻一个「🕒 最近使用」模块：

- **数据来源**：在 Tab It 内点击任意书签打开后会自动记入；按 `openedAt` 倒序排列。
- **展示数量**：默认 N=8。鼠标 hover 头部显示「N=xx」按钮，点击可改为 1 ~ 100 之间的任意整数。
- **折叠**：hover 头部显示折叠 ▸ 按钮。
- **清空**：hover 头部显示 🗑 按钮，二次确认后清空全部最近使用记录（不删书签本身）。
- **合并浏览器历史**（v0.6+ 可选）：开启后会叠加 `browser.history.search()` 的全局历史；
  需要 `history` 权限，**默认关闭** —— 隐私敏感，由你主动开启。
- 已被删除的卡片会自动从最近使用里跳过。
- 在搜索模式下不渲染（避免和搜索结果叠加）。

### 6.2 当前分类的内容

进入某个分类后，主内容区结构如下：

```
最近使用
─────────────────────
当前分类（compact header）
  └─ 文件夹（直接子文件夹）
  └─ 书签（直接书签 + 末尾的 + 卡片）
─────────────────────
子分类 1 / 子分类 1 的孙分类（full header）
─────────────────────
子分类 2（full header）
...
```

- **DFS 递归展开**：当前分类的所有后代分类（不止直接子级）都会作为独立 section 渲染，方便一眼浏览整个分支。
- **section 折叠**：每个 section header 提供 ▸ 折叠按钮（hover 显示）。
  - 根 section（compact header）：默认**展开**。
  - 子 section（full header）：默认**折叠**，点击 ▸ 按需展开。
  - 切换分类时所有 section 回到默认态（key 含 activeCategoryId 强制 remount）。
- **空文件夹**：会显示「空文件夹」提示，保留目录结构可见。

### 6.3 添加书签

两种入口：

1. 根 section 的书签网格末尾有一个 **+ 卡片**。
2. 子 section header hover 时右侧有 **+** 按钮。

依次输入：URL → 标题（默认值是 URL）→ 创建。

也可以使用浏览器工具栏 popup（§11）"添加当前页面"功能。

### 6.4 卡片操作

每张卡片：

- **整张点击**：在新标签页打开 URL。会自动记入「最近使用」。
- **拖拽排序**：按下 5px 阈值后开始拖拽（≤ 5px 视为单击），仅在同一分类内排序。
- **菜单（hover 右上角 ⋯）**：编辑、添加/编辑备注、**✨ 相关阅读**（§9.5）、删除。

#### 卡片头部紧凑指示器（v0.11+）

域名行右侧会显示几种紧凑指示器（按显示优先级）：

| 指示器 | 意义 |
|---|---|
| 📄（绿色徽标） | §9.3 已抓取该网页正文 · hover 提示「可参与 AI 语义搜索」 |
| `#前端` 紫色 chip | 该卡片的 tags；点击 chip → 触发 `#tag` 筛选（§5.3 / §8.5） |
| `+N` 灰色 chip | 该卡片还有 N 个 tag 未显示，hover 看全部 |

#### 编辑模式

进入编辑模式后：

- **标题** 与 **URL** 两个字段可同时修改；都不能为空。
- 左上角的图标也可点击进入 IconPicker 改图标。
- **Enter 保存** / **Esc 取消**。
- 完全没有改动时直接退出，不写库。

#### 备注

- 已有备注：卡片底部展示备注（最多 2 行 line-clamp），点击编辑。
- 无备注：hover 卡片时底部出现 +「添加备注」按钮。
- 备注通过浏览器原生 prompt 输入；输入空字符串则清除备注。
- v0.17+ AI 自动备注功能可批量生成（§9.4）。

#### 删除

确认弹窗里会显示书签标题，避免误删。

### 6.5 文件夹卡片

子文件夹会作为一种特殊卡片显示在「文件夹」区。点击进入子分类（与点击侧栏同步选中）。

### 6.6 全库搜索 / 标签筛选 / 语义搜索

> 现在已经实装在顶栏搜索框（§2.1）—— 输入任意关键字即可全库搜书签；
> `#标签名` 走标签精确筛选；`@ai 关键字` 走语义搜索（需先生成 embedding）。
>
> 搜索结果视图：按 url 去重（同一站点被多个分类收藏时只显示代表那张），
> 每张卡片底部显示 `📂 分类路径 [+N 其他副本]` 让你知道这条来自哪里、还有没有副本。

## 7. AI 助手浮窗（v0.8+）

> AI 是**完全 opt-in** 的可选能力。不配置 Provider 时 FAB 灰色，
> 浮窗也只在引导你配置；正常使用书签管理不受任何影响。

### 7.1 右下角 ✨ FAB 与浮窗

| 状态 | 视觉 | 点击行为 |
|---|---|---|
| 未配置 | 灰色 ✨ | 打开浮窗 + 自动落到「⚙ 设置」Tab，引导添加 Provider |
| 已配置 | brand 色 ✨ | 打开浮窗（恢复上次状态） |
| 浮窗显示中 | FAB 隐藏 | 避免双入口 |
| 有被动建议（§9.6） | 右上小红点 | 点击 → 自动打开「🗂 整理」Tab + 预填范围 |
| AI 处理中 | 呼吸光晕 | （目前仅占位，未来阶段任务跑时会激活） |

**快捷键**：`Cmd/Ctrl + J` 全局唤起 / 隐藏浮窗（与 Notion AI 对齐）。

### 7.2 浮窗几何

- **默认尺寸**：380 × 520。最小 280 × 360。
- **默认位置**：右下角，距 FAB 稍上方。
- **拖动**：整个 header 都是拖动区。
- **缩放**：右边缘 / 下边缘 / 右下角 三处手柄。
- **双击 header**：切换最大化（70vw × 80vh，居中 + 半透明遮罩）。
- **右键 header**：恢复默认位置和尺寸。
- **最小化**：右上 ─ 按钮 → 右下角小 chip 显示当前活跃 tab 标题；ESC **不**关闭。
- **持续存在**：失焦 / 点击主区 都**不会**关闭浮窗（这是与传统 modal 的核心区别）。
- **位置 / 尺寸 / 当前 Tab 列表**：全部持久化，重启浏览器后恢复。
- **底部状态栏**（v0.14+）显示当前 chat 任务实际指向的 Provider：
  - `✓ Local`（绿）：开了"优先本地" + Chrome 内置 Gemini Nano 可用
  - `☁ Cloud`（蓝）：走配置的远程 Provider
  - `⚠ Local 不可用`（橙）：开了优先本地但浏览器不支持 → 自动回落到云端

### 7.3 浮窗内的 Tab 体系

```
[💬 对话] [🗂 整理] [🏷 标签] [⚙ 设置] [+]
```

- 每个 Tab 一份独立 state（messages / plan 等），随浮窗一并持久化。
- 关闭浮窗 / 切换 Tab 都**不会**丢失任务进度，下次打开恢复到对应阶段。
- 「+」按钮可新建任意类型 Tab，例如多开 💬 对话 Tab 同时挂多个会话。
- v0.20+ 每个 tab hover 时多出一个 **⤴ 分离**按钮（§7.4 多浮窗）。

### 7.4 多浮窗 / 分离 Tab（v0.20+）

```
主浮窗 tab bar：[💬 React 性能 ⤴] [🗂 整理 ✕] [+]
                              ↑
                              已分离的 tab 上有 ⤴ 标记

副浮窗（独立可拖）：
  ┌────────────────────┐
  │ ⠿ 💬 React 性能  副 ─ ✕│
  ├────────────────────┤
  │  对话流...         │
  └────────────────────┘
```

- hover 主浮窗任意 tab → 出现 ⤴ 按钮 → 点击把它"分离"成独立浮窗。
- 副浮窗可独立**拖动 / 缩放 / 最小化**；header 是 fuchsia 色调 + 「副」徽标做视觉区分。
- 副浮窗最小化时**保留在原位置**变小 chip（Slack 浮动通话窗模式）。
- 副浮窗与主浮窗看的是**同一份数据**：在任一边编辑会实时同步到另一边。
- 关闭副浮窗 ≠ 删除 tab。tab 数据仍在主浮窗里，可以重新切换查看。
- 想完全删 tab 用主浮窗的 tab ✕（同时会清理对应副浮窗）。
- 副浮窗位置 / 尺寸独立持久化，重启后恢复。
- 不支持最大化（多个副浮窗都最大化会互相遮挡）。

## 8. AI 设置（⚙ Tab）

进入路径：**FAB ✨ → ⚙ 设置**。

### 8.1 总开关与隐私

| 开关 | 默认 | 说明 |
|---|---|---|
| 启用 AI 功能 | OFF | 关闭时所有 AI 入口（FAB / 浮窗 / popup ✨）都不工作；添加第一个 Provider 时自动打开 |
| 匿名模式 | ON | 发送给 AI 时**只发域名**，不发完整 URL（产品红线） |
| 操作前显示成本估算 | ON | 每次 AI 操作前先确认本次大约消耗多少 tokens / 费用 |
| 优先使用浏览器内置 AI | OFF | 可用时优先走 Chrome 138+ 内置 Gemini Nano；仅 chat 任务生效 |
| 被动整理建议（§9.6） | ON | 累计新增 ≥10 条 + 距上次 ≥7 天 时 FAB 红点提示 |
| 抓取完成后提示生成备注（§9.4） | OFF | crawler 完成后弹 toast 引导去 AI 自动备注 section |

### 8.2 AI Provider

支持 18 个预设（5 分组）+ 完全自定义：

| 分组 | 代表 |
|---|---|
| 🇨🇳 国内（中文友好） | DeepSeek · Moonshot Kimi · 智谱 GLM · 阿里通义千问 · 字节豆包 · 零一万物 Yi · MiniMax · 百川 |
| 🌎 国外 | OpenAI · Azure OpenAI · Groq ⚡ · Cerebras ⚡ · Together AI |
| 🔄 聚合 / 中转 | OpenRouter · SiliconFlow |
| 💻 本地部署 | Ollama · LM Studio · vLLM |
| 🧪 实验性 | Chrome 内置 Gemini Nano |

**操作**：
- 点「+ 添加 Provider」展开预设下拉 / 自定义表单
- 必填：name · baseURL · apiKey（本地部署的 Ollama 等可留空）· model
- 可选：embeddingModel（仅在 §9.2 / §9.5 等需要 embedding 时用到）
- 添加后会自动用作所有任务的默认 Provider
- 每条 Provider 右侧有「测试连接」按钮，发送 hi 验证

**API Key 安全承诺**：
- 仅存 `chrome.storage.local`，永不出包
- 不进 `storage.sync`（即使你浏览器开了同步也不上传）
- 不进 export json 文件（分享 / 备份给别人时不会泄漏）

### 8.3 任务路由（≥ 2 个 Provider 时显示）

可以为不同任务指定不同 Provider，例如：

- 对话 / 总结 → 用强模型（GPT-4o / DeepSeek-Reasoner）
- 整理 / 分类 / 标签 → 用便宜模型（gpt-4o-mini / DeepSeek-Chat）
- Embedding → 用专门的 embedding 模型

### 8.4 Embedding 管理（§9.2 语义搜索的基础）

```
书签总数: 423            已索引: 320 ✓
待补缺: 103 ⚠            内容已变 (stale): 5 ⚠
使用模型: text-embedding-3-small

[✨ 补缺 (108)]  [全部重生成]                [清空]
```

**操作**：
- **✨ 补缺**：仅为缺失 / 内容变化的卡生成（推荐；增量、低成本）
- **全部重生成**：清空整库后从头跑（切换 embedding 模型 / 大改后用）
- **清理孤立**（仅在有孤立行时出现）：删除已被删除卡片对应的 embedding 行
- **清空**：完全重置，语义搜索回退到 substring

> 当 §9.3 抓取了网页正文后，再次「补缺」会把正文也纳入 embedding 输入，
> 极大提升语义召回质量；这是 §9.7 RAG 问答的基础。

### 8.5 内容抓取（§9.3 的入口）

详见 §9.3。第一次使用前会强制弹出**隐私说明弹窗**：
- 明确说"将下载 N 个网页内容到本机 IndexedDB"
- 列出四条承诺：不上传 / 不带 cookie / 仅主动操作 / 可撤回
- 必须勾「我已了解并同意」才能开始

### 8.6 AI 自动备注（§9.4 的入口）

详见 §9.4。需要先 §9.3 抓取过正文。

### 8.7 整理质检（§9.8 的入口）

详见 §9.8。可独立使用，但 embedding 相似度检测需要 §9.2 已生成。

## 9. AI 能力一览

> 所有能力都遵循同一套设计原则：
> - **opt-in**：默认关闭，必须主动配置 / 操作
> - **隐私优先**：默认匿名模式（仅域名）
> - **可撤销**：写库操作有快照 + 60s 撤销窗（如 §9.1 AI 整理）
> - **成本透明**：每次操作前显示估算

### 9.1 AI 整理书签（🗂 Tab）

适合**新装 / 一堆没分类的浏览器导入**之后用：

1. **配置阶段**：选范围（全部 / 顶层未分类项 / 某分类）+ 风格倾向（工作 / 学习 / 生活 / 自由发挥）
2. **成本估算**：明确显示发送条数 / token / ¥ 估算
3. **执行**：分批 100 条调 LLM，可中途取消
4. **预览**：按 section 分组显示「+ 新建分类」「↻ 移动书签」「🗑 删除空分类」；单条可 ✓/✗
5. **应用**：写入前自动 snapshot；toast 提示「60s 内可撤销」
6. **撤销窗**：点 toast 上的「↶ 撤销整理」即可完整回滚

### 9.2 AI 语义搜索（@ai 模式）

```
顶栏搜索框输入：@ai 关于 React 性能优化的文章
↓
主区显示：
  [AI 语义搜索: 关于 React 性能优化的文章   · 8 命中 + 12 兜底]  ✕
  ┌────────┐ ┌────────┐ ┌────────┐ ...
  │ 卡片1  │ │ 卡片2  │ │ 卡片3  │
  │   72%  │ │   65%  │ │   48%  │   ← 角标显示相似度
  └────────┘ └────────┘ └────────┘
  · 关键字兜底匹配（embedding 未覆盖）─────
  ...
```

- **前提**：§8.4 已生成 embedding（点过「✨ 补缺」）
- **过程**：把 query embedding 化后做 cosine top K；debounce 350ms 避免抖动
- **兜底**：embedding 未覆盖但 substring 命中的卡片作为次级结果展示
- **不调外部服务**做检索本身：仅本地余弦计算，零延迟（仅 query 那一次 embed 调 API）

### 9.3 网页内容抓取（⚙ → 内容抓取）

把已收藏网页的正文抓回本地，作为 §9.7 RAG 问答和 §9.2 增强语义搜索的基础。

**首次使用**：必须先在弹窗里勾「我已了解并同意」隐私说明（一次性，可撤回）。

**范围选择**：
- 仅未抓取过的（推荐 · 增量）
- 重试上次失败的
- 全部书签（覆盖已索引）
- 按某顶层分类

**抓取流程**：
- fetch + `credentials: 'omit'`（不带登录态）
- 30s 超时；Mozilla Readability 提取正文 → 截断到 8000 字符
- 写入本机 IndexedDB；并发 3，失败的卡片标记 status='failed'，下次「重试失败」可重抓
- 进度条显示「正在抓取的标题」

**抓完后**：
- 卡片右上角出现 📄 绿色小角标
- 「补缺 embedding」按钮会自动提示"内容已变 N 张"，建议立即补 → 让正文也纳入语义检索

### 9.4 AI 自动备注（⚙ → AI 自动备注）

需要先 §9.3 抓取过正文。

- 默认只处理"已抓正文且没写备注"的卡（**不覆盖**用户已写的内容）
- 单条调用 LLM 给一句话摘要（≤ 25 字），并发 2
- 写入 `card.description`
- 「抓取完成后提示生成备注」开关：crawler 完成后弹 toast 引导来这里（不静默调）

### 9.5 相关阅读（卡片菜单 → ✨ 相关阅读）

- 任意卡片右上 ⋯ → 「✨ 相关阅读」
- 弹层显示**纯本地 cosine** 算出的 top 3 相似卡（minScore 0.4，比 RAG 更严，质量优先）
- 不调任何 LLM，零成本零延迟
- 不引入外部数据，仅复用你自己的书签

### 9.6 被动整理建议（§5.2 红点）

不打扰式提醒：

- 当累计新增书签 ≥ 10 条 + 距上次提示 ≥ 7 天 时，FAB 出现红点
- 点击红点 → 自动打开「🗂 整理」Tab + 预填 "顶层未分类项" 范围
- OrganizeTab 顶部也会有 💡 横幅文案
- 在 ⚙ 设置开关可关闭

### 9.7 RAG 问答 / 对话（💬 Tab）

```
[使用 deepseek-chat]  [📚 RAG · 开]                [导出 .md] [清空]
─────────────────────────────────────────────────────
🙋 上周收藏的那篇讲 React useReducer 的怎么说

✨ useReducer 适合"多个相关 state 一起变" [1] 和"下一个 state 依赖前一个" [2]
   的场景。性能上注意…
   ┌─────────────────────────────────┐
   │ 参考来源 (2)                    │
   │ [1] Dan Abramov: 何时用 useReducer  72% │
   │ [2] React 官方文档: useReducer API 65% │
   └─────────────────────────────────┘

[输入消息，Cmd+Enter 发送                              ] [发送]
```

**关键点**：

- 顶部 **📚 RAG** 开关：必须有已抓取的正文（§9.3）才能启用
- 开启后：每次提问先做向量检索 top K → 把片段作为 system context → 流式回答
- 答案中 `[1] [2]` 与底部「参考来源」列表一一对应；点击参考项打开原网页
- 召回失败也会回答，但 system prompt 会引导模型说明"本回答不来自您的收藏"
- 关闭 RAG = 纯通用对话（不读你的书签）
- **多对话**：浮窗 tab bar「+」可新建任意数量 💬 对话；tab 标题自动从首条提问生成
- **导出 .md**：完整对话 + 引用脚注下载
- **流式 / 中止**：发送中按「停止」会保留已生成的部分

### 9.8 整理质检（⚙ → 整理质检）

扫描全库找出三类问题：

| 类别 | 判定方法 |
|---|---|
| 🔴 **失效** | HEAD 请求 4xx/5xx / 网络错 / 15s 超时（部分服务器不响应 HEAD 时降级 GET） |
| 🟡 **疑似重复** | URL 完全一致 → exact_url 组；URL 不同但 embedding cosine ≥ 0.9 → similar_content 组 |
| 🔵 **长期未访问** | `updatedAt` < 现在 - 6 个月 |

操作：

- 三色分组列表，每张卡可单独勾选
- 重复组内默认提示「保留」第一项；其余建议处理
- 底部：「🗑 批量删除」/ 选目标分类 + 「📦 批量归档」

### 9.9 AI 打标签（🏷 Tab）

```
[✨ 批量打标签] | [🏷 标签管理]

— 批量打标签 wizard —
范围：✓ 仅未打标签的书签（推荐）
      ○ 全部书签（覆盖已有 tags）
      ▸ 按某个顶层分类打标签

[✨ 让 AI 打标签]
↓ 进入 estimate / running / preview / applying / done 状态机
预览：每张卡片显示 oldTags（划线）→ newTags（紫色）；可勾选 / 编辑 / 还原
应用：写入 IndexedDB
```

**标签管理面板**：
- 列出全库所有 tags + 出现次数
- 改名 / 合并（≥ 2 个选中）/ 删除
- 点击 tag 名 → 触发 `#tag` 筛选

### 9.10 AI 添加书签（popup → ✨）

详见 §11 popup。

## 10. 数据持久化与隐私

### 10.1 存储位置

| 数据 | 存储 | 容量 |
|---|---|---|
| 分类、书签、用户设置 | `chrome.storage.local` 的 `tabit:*` keys | 5MB 共享 |
| 最近使用记录 | `chrome.storage.local` | 同上 |
| AI 设置（含 apiKey） | `chrome.storage.local` 的 `tabit:ai-settings` | 同上 |
| 浮窗几何 / Tab 列表 | `chrome.storage.local` 的 `tabit:ai-panel` | 同上 |
| 副浮窗几何 | `chrome.storage.local` 的 `tabit:ai-secondary-panels` | 同上 |
| Embedding 向量库 | IndexedDB `tabit-embeddings` | 浏览器配额（GB 级） |
| 网页正文索引 | IndexedDB `tabit-pages` | 同上 |
| 当前选中的搜索引擎 | `localStorage` | 仅当前域 |

### 10.2 容量上限

- `chrome.storage.local` 整体上限 **5MB**：
  - 壁纸 2MB、图标 100KB 都有限制（base64 占空间）
  - AI 设置体积小（< 10KB）
- IndexedDB 容量很大（数 GB），千张书签的 embedding（每个 6KB float32）+ 正文（每个 ≤ 8KB）都装得下

### 10.3 备份策略

强烈建议定期使用「**导出配置文件**」做本地备份，特别是在：

- 进行大规模整理前
- 切换设备前
- 升级扩展版本前
- 用 AI 整理（§9.1）前（虽然内部有 60s 撤销窗，但 toast 关闭就清了）

> 导出文件**不包含** AI 设置 / Embedding / 正文索引 / Tag 元数据。这些是本机派生数据，
> 真要迁移：导出书签 → 新设备导入 → 重新配置 AI Provider → 重新生成 embedding / 抓取。

### 10.4 卸载扩展会发生什么

`chrome.storage.local` + IndexedDB 的数据都会随扩展卸载一并清除，且**不可恢复**。
重新安装后是空白状态，需要重新导入备份或浏览器书签。

### 10.5 隐私红线总结

- Tab It 是**完全本地**的扩展，**不会上传到任何服务器**（除非你主动调用某个 AI Provider，那时仅发送给你配置的那个 Provider）。
- 发送给 AI 的数据**默认匿名**：只发域名，不发完整 URL（产品红线，§9.x 各服务都遵守）。
- 「网页内容抓取」(§9.3) 是**唯一**会真正访问外部网站 fetch HTML 的能力 —— 默认完全关闭，
  必须用户主动勾隐私同意才会开始；fetch 时显式 `credentials: 'omit'` 不带登录态；抓到的内容仅写本机 IndexedDB。
- **API Key 永不出包**：不进 storage.sync、不进 export json。
- 「从浏览器导入书签」需要 `bookmarks` 权限，仅用于读取你的书签数据，不上报。
- 「最近使用 → 合并浏览器历史」可选功能需要 `history` 权限，默认关闭。
- 网页搜索是直接 `window.open` 跳转到对应搜索引擎，**不经过 Tab It 中转**，搜索词不留任何记录。

## 11. popup（浏览器工具栏图标）

点击浏览器工具栏的 Tab It 图标弹出小窗（360px 宽）。两个主操作：

### 11.1 打开 Tab It 新标签页

一键打开 `chrome://newtab`（Firefox 用 `newtab.html`）。

### 11.2 添加当前页面

把当前活跃 tab 的 title / url 加为 Tab It 书签：

- **预览区**：favicon + 域名
- **标题**：可改（默认是浏览器原标题）
- **备注**：可填可不填
- **标签**：用空格 / 逗号 / 顿号分隔；下方实时显示解析后的 chips
- **添加到**：从扁平的分类下拉选目标分类（默认是 newtab 上次激活的分类）
- **✨ AI 建议**（v0.12+，仅 AI 已配置时显示）：
  - 一键调用 LLM 生成 `suggestedCategory` / `description` / `tags` 三个建议
  - 只覆盖你**没主动改过**的字段（dirty 跟踪）
  - 错误时显红色横条 + 重试链接
  - 用户改过的 category 字段被 AI 建议另一个时，会以次级链接「✨ AI 建议: xxx」展示让你主动采用
- **提交**：「添加到 Tab It」；同 URL 已存在时友好提示「该分类下已有此 URL」

> popup 不接入浮窗范式 —— 浏览器弹窗本身就是浮层，再嵌套体验奇怪。

## 13. 故障排查

### 13.1 Firefox 卡在「加载中…」

历史问题，已修复（v0.1.0+）。如仍遇到：

- 确认使用 `pnpm build:firefox` 而不是直接用 chrome 包。
- 在 `about:debugging` 重新临时载入。
- 检查浏览器控制台是否有 `chrome is not defined` 错误，若有请提交 issue。

### 13.2 侧栏宽度拖完松手又跳回去

历史问题，已修复（v0.1.0+）。根因是首次 mount 时 `useEffect` 把默认值写覆盖了已存的真实值。

### 13.3 从浏览器导入后看不到分类

- 检查浏览器原生书签是否真的有内容（`chrome://bookmarks` / `about:bookmarks`）。
- 看看左侧是否在「批量管理」模式或「重命名」模式。
- 强制刷新一下新标签页（`Ctrl+R` / `Cmd+R`）。

### 13.4 自定义图标 / 壁纸上传报「图片过大」

- 图标上限 100KB，壁纸上限 2MB。
- 解决：在线压缩工具压缩后再上传；或改用 URL 形式（不占本地容量）。

### 13.5 删除分类后想恢复

目前**没有撤销**功能（已在 ROADMAP 中规划）。强烈建议依赖「导出配置文件」做定期备份。
注意：AI 整理（§9.1）单独有 60s 撤销窗，但仅限那次整理本身。

### 13.6 AI 相关常见问题

| 现象 | 排查 |
|---|---|
| FAB 是灰色 | 还没配置 Provider，点 FAB 进 ⚙ 设置添加一个 |
| 「请先配置 AI Provider」 | 同上 |
| Chrome 内置 AI 返回乱码（中文） | Gemini Nano 仅支持 en/es/ja，**不擅长中文**；ChatTab 顶部有橙色警告条 + 一键切换到远程 Provider |
| 测试连接失败 「HTTP 401」 | apiKey 错或没填 |
| 测试连接失败 「HTTP 404 / model not found」 | model 名拼错；豆包等服务要填 endpoint id 而不是模型名 |
| @ai 搜索没结果 | 还没生成 embedding，进 ⚙ 设置点「✨ 补缺」 |
| RAG 开关 disabled | 还没抓取任何网页正文，进 ⚙ 设置 → 内容抓取 |
| RAG 回答说"未找到相关内容" | 召回为空（minScore=0.25 过滤掉了所有低分文档）；可能你的书签库里确实没这个主题 |
| 卡片右上没有 📄 角标 | 该卡未被 §9.3 抓取过；可能 URL 不是 http(s)（如 chrome://），或抓取失败了 |
| 整理质检里没有 "🟡 内容相似" | 需要 §9.2 已为参与的卡片生成 embedding；先在 ⚙ 补缺 |

### 13.7 撤回隐私同意 / 关闭所有 AI

- **临时关闭**：⚙ 设置 → 通用 → 关闭「启用 AI 功能」开关 → 所有 AI 入口立即不工作（FAB 变灰）
- **撤回网页抓取同意**：⚙ 设置 → 内容抓取 → 「撤回同意」（本地已抓取的内容保留；不会再发起新抓取）
- **删除 AI 数据**：⚙ 设置 → Embedding 管理 → 清空；内容抓取 → 清空
- **完全卸载**：浏览器扩展管理页移除 Tab It → 所有本地数据（含 AI 设置 / apiKey / IndexedDB）一并清除

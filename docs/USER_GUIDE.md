# Tab It 使用指南

> 一个替代浏览器新标签页的书签整理工具，支持 Chrome 与 Firefox。
>
> 本文档同时是「顶栏 → 帮助」按钮里弹窗的内容来源，会随产品版本同步更新。

---

## 1. 初次使用

### 1.1 安装扩展

#### Chrome / Edge / 其他 Chromium 内核浏览器

1. 下载并解压 `tab-it-<version>-chrome.zip`（或自行执行 `pnpm build` 后用 `.output/chrome-mv3` 目录）。
2. 浏览器地址栏输入 `chrome://extensions`，打开右上角「**开发者模式**」开关。
3. 点击「**加载已解压的扩展程序**」，选择上一步的目录。
4. 安装完成后，**新开一个标签页**即可看到 Tab It。

#### Firefox

1. 下载 `tab-it-<version>-firefox.zip`（或自行执行 `pnpm build:firefox`，输出在 `.output/firefox-mv2`）。
2. 浏览器地址栏输入 `about:debugging#/runtime/this-firefox`。
3. 点击「**临时载入附加组件**」，选择 zip 包内的 `manifest.json`（或 `firefox-mv2/manifest.json`）。
4. 临时安装的扩展会在浏览器关闭后失效，正式发布版可通过 AMO（addons.mozilla.org）签名后永久安装。

> **跨浏览器差异**：Tab It 在 Chrome 使用 MV3、Firefox 使用 MV2，并通过 `wxt/browser` 抹平 `chrome.*` API 的 Promise 差异，使用体验完全一致。

### 1.2 首次打开看到什么

- **顶栏**：左侧 Logo、中间网页搜索框、右侧「帮助」与「设置」两个按钮。
- **左侧分类栏**：默认空白，会提示「还没有分类」。
- **主内容区**：未选中分类时为空。

### 1.3 第一步建议：导入浏览器现有书签

打开「设置（⚙）」→「数据管理」→「**从浏览器导入书签**」，会按浏览器原有的文件夹层级把所有书签合并进来，保留分类结构。重复点击会做**幂等合并**：已存在的书签按 ID 更新，不会产生重复。

## 2. 顶栏

```
[ Tab It ]   [ 🔍 网页搜索框 ]                       [ ? ]  [ ⚙ ]
                                                     帮助    设置
```

### 2.1 网页搜索框

由于扩展接管了新标签页，浏览器原生地址栏的「在新标签搜索」体验失效，所以这里补了一个等价能力：

- **切换搜索引擎**：点击搜索框左侧的引擎图标（▾），可在 **Google / Bing / 百度 / DuckDuckGo** 四个之间切换；选择会持久化在 `localStorage`，下次进入沿用。
- **回车提交**：在「**新标签页**」打开搜索结果（不会覆盖当前页）。
- **清空**：右侧 ✕ 按钮一键清空输入。
- **本地搜索 vs 网页搜索**：本搜索框只发起**网页搜索**；要在自己的书签里搜索，请使用 V0.2 即将上线的「全库搜索」（路线图）。

### 2.2 帮助按钮

齿轮按钮**左侧**的问号图标即是帮助按钮，点击会打开本文档的弹窗。

- **关闭弹窗**：点击右上角 ✕、点击遮罩区域、或按 `Esc` 键。
- **滚动**：内容超过弹窗高度时，弹窗内部独立滚动，不影响背景页面。

### 2.3 设置按钮（齿轮 ⚙）

齿轮按钮是一个**气泡菜单**，点击展开两个二级入口：

| 入口 | 作用 |
| --- | --- |
| 数据管理 | 导入 / 导出 / 从浏览器同步 |
| 样式管理 | 主题、自定义壁纸 |

气泡菜单点击外部或按 `Esc` 自动关闭。

## 3. 数据管理

进入路径：**齿轮 → 数据管理**。弹窗内提供 3 个原子操作。

### 3.1 从浏览器导入书签

将当前浏览器（Chrome 或 Firefox 原生书签）一键合并到 Tab It：

- 按浏览器的**文件夹层级**还原为 Tab It 的「分类 / 子分类」结构。
- 已存在的书签按 ID **幂等合并**，不会产生重复。
- 适合**初次安装**或**新增了浏览器原生书签后**同步过来。

### 3.2 导入配置文件

选择一份 `tab-it-export-*.json` 文件（自己导出的 / 别人分享的）。导入前会弹出二次确认，让你选择「**合并**」还是「**替换**」：

| 模式 | 行为 | 适合场景 |
| --- | --- | --- |
| 合并到现有数据（**推荐**） | 保留本地分类与书签，按 ID 合并；同 ID 取较新者，新数据追加到末尾 | 多设备同步、合并别人的分享 |
| 替换全部数据（**慎用**） | 清空本地所有分类与书签，并使用文件中的设置覆盖当前主题/布局 | 完全恢复到某次导出时的状态 |

> 文件需为 JSON 且包含 `categories` / `cards` 字段，否则会提示「未识别到分类或书签数据」。

### 3.3 导出配置文件

将当前所有分类、书签、设置打包为 JSON 文件下载到本地，文件名格式：

```
tab-it-export-YYYY-MM-DD.json
```

可用于备份、跨设备迁移、给同事分享自己的「精选书签集」等。

> 导出文件**不包含**最近使用记录、`localStorage` 里的搜索引擎偏好，仅包含分类、书签、用户设置三大块。

## 4. 样式管理

进入路径：**齿轮 → 样式管理**。

### 4.1 主题

三种主题任意切换：

| 主题 | 行为 |
| --- | --- |
| ☀️ **明亮** | 强制亮色 |
| 🌙 **黑暗** | 强制暗色 |
| 🖥️ **跟随系统** | 默认。跟随操作系统外观偏好（`prefers-color-scheme`） |

### 4.2 自定义背景

#### 6 个预设壁纸

| 名称 | 风格 |
| --- | --- |
| 默认 | 回退到浅紫渐变（`global.css` 内置） |
| 极光 | 紫 → 蓝 → 青三段渐变 |
| 暮色 | 橙 → 粉双色渐变 |
| 深海 | 青 → 蓝双色渐变 |
| 林荫 | 绿色双段渐变 |
| 午夜 | 深蓝 → 深紫双段渐变 |

点击缩略图即时切换，已选项右下角会有 ✓ 标记。

#### 自定义图片 URL

把任意图片链接粘到输入框（支持 `https://` 或 `data:image/...` 两种），按 **Enter** 或点「应用」即可。

#### 从本地上传

- 支持 PNG / JPG / WEBP / GIF 等浏览器认识的图片格式。
- **文件大小上限 2MB**。超出会拒绝并提示原因。
  - 原因：本地壁纸以 base64 存进 `chrome.storage.local`，该容器单条上限 ≈ 5MB，预留余量给其他数据。
- 上传后立即生效，且会持久化。

#### 清除背景

如果当前已设置了自定义壁纸，按钮区会出现红色「✕ 清除背景」按钮，一键回退到「默认」状态。

## 5. 左侧分类栏

分类栏是 Tab It 的核心组织方式，支持**多层嵌套**、**拖拽排序**、**批量管理**等。

### 5.1 顶部按钮一览

```
[ ☰ 分类 ]                             [ ⇕ ] [ ☑ ] [ + ] [ ‹ ]
                                       展开/  批量  新建  收起
                                       折叠   管理  顶层  侧栏
                                       全部
```

| 按钮 | 行为 |
| --- | --- |
| **展开/折叠全部** | 一键展开 / 一键折叠所有有子分类的节点。当前没有任何子分类时禁用。 |
| **批量管理** | 进入多选模式（详见 5.5） |
| **+** | 新建一个**顶层**分类，会弹 prompt 输入名称 |
| **‹** | 收起整个侧栏到 40px 窄条 |

### 5.2 单条分类的交互

每行分类从左到右依次是：

1. **展开三角 ▶**：当且仅当该分类有子分类时显示，点击切换展开/折叠
2. **图标**：默认顶层 📁 / 子级 📂；点击进入「图标选择器」修改（详见 5.3）
3. **名称**：双击进入重命名（Enter 保存 / Esc 取消）
4. **数量**：递归统计该分类下（含所有子级）的书签总数
5. **+（hover 显示）**：在该分类下新建一个**子分类**，新建后自动展开父级
6. **🗑（hover 显示）**：删除该分类。删除前弹出确认；如果有子分类会提示「**及其所有子文件夹和书签**」

> **点击行为**：单击分类名称会进入该分类（主内容区刷新），同时如果有子分类会自动展开。

### 5.3 图标选择器（IconPicker）

点击任何分类前的图标，弹出 3-Tab 选择器：

- **Emoji**：从 64 个常用 emoji 中点选。
- **链接**：粘贴图片 URL，输入框下方实时预览，确认后点「应用」。
- **上传**：本地选图（**100KB 上限**，比壁纸更严格，因为图标会大量存储）。
- **重置**：一键恢复默认 emoji（顶层 📁 / 子级 📂）。

弹层会自动避让边界：靠右展开会自动右对齐；下方空间不足会自动向上展开。

### 5.4 添加子分类

两种方式：

1. **行内 +**：鼠标 hover 到目标父分类，点右侧出现的 +。
2. **拖拽嵌入**：把任意分类拖到目标分类**中部 40% 区域**（详见 5.6 拖拽规则），会成为目标的子节点。

### 5.5 批量管理

点击顶部「批量管理」按钮（☑ 图标）进入多选模式：

- 复选框只对**顶层分类**可用，子分类不能单独勾选（按整棵子树一起处理，避免遗留孤儿节点）。
- 顶部按钮变为：「✓ 全选 / ✕ 取消全选」、「🗑 批量删除」、「完成」。
- 删除前会提示：「确定删除 X 个分类（含子分类）、Y 个卡片吗？」
- 退出多选只需点「完成」。

### 5.6 拖拽分类（跨层级）

整行都是拖拽手柄。拖动有 **6px 距离阈值**，避免误触发（点击不会触发拖拽）。

拖到目标行时，根据**指针在目标行的纵向位置**决定语义：

| 区域 | 视觉指示 | 语义 |
| --- | --- | --- |
| 上方 30% | 顶部出现一条 brand 色横线 | **before**：插入到目标**之前**（同层） |
| 中间 40% | 整行出现 brand 色环线 + 浅色背景 | **inside**：成为目标的**子节点**（自动展开父级） |
| 下方 30% | 底部出现一条 brand 色横线 | **after**：插入到目标**之后**（同层） |

**约束**：

- 不能把分类拖到自己的后代上（循环引用），系统会自动取消该次拖拽。
- 重命名 / 多选模式下禁用拖拽，避免冲突。

### 5.7 折叠 / 展开整个侧栏

点顶部 ‹ 按钮可以把整个侧栏收起到 40px 窄条，腾出空间给主内容。

收起态会：

- 显示展开 › 按钮。
- 显示**前 6 个**顶层分类的图标，点击单个图标会自动展开侧栏并跳转到该分类。

### 5.8 拖拽调整侧栏宽度

侧栏的右边缘有一个 **6px 拖拽热区**（hover 时会显示 2px brand 色细条）：

- **拖拽**：按住即可调整宽度，范围 **180 ~ 480px**。
- **双击**：恢复默认宽度（240px）。
- 折叠状态下隐藏热区（折叠后宽度固定 40px）。
- 宽度会持久化在 `tabit:settings` 中，下次打开沿用。

## 6. 主内容区

### 6.1 最近使用模块

只要在分类视图下，分类内容**上方**会常驻一个「🕒 最近使用」模块：

- **数据来源**：在 Tab It 内点击任意书签打开后会自动记入；按 `openedAt` 倒序排列。
- **展示数量**：默认 N=12（不同版本可能略有差异）。鼠标 hover 头部显示「N=xx」按钮，点击可改为 1 ~ 100 之间的任意整数。
- **折叠**：hover 头部显示折叠 ▸ 按钮。
- **清空**：hover 头部显示 🗑 按钮，二次确认后清空全部最近使用记录（不删书签本身）。
- 已被删除的卡片会自动从最近使用里跳过。
- 在搜索模式下不渲染（避免和搜索结果叠加）。

### 6.2 当前分类的内容

进入某个分类后，主内容区结构如下：

```
最近使用
─────────────────────
当前分类（compact header）
  └─ 文件夹（直接子文件夹）
  └─ 书签（直接书签 + 末尾的 + 卡片）
─────────────────────
子分类 1 / 子分类 1 的孙分类（full header）
─────────────────────
子分类 2（full header）
...
```

- **DFS 递归展开**：当前分类的所有后代分类（不止直接子级）都会作为独立 section 渲染，方便一眼浏览整个分支。
- **section 折叠**：每个 section header 提供 ▸ 折叠按钮（hover 显示）。
  - 根 section（compact header）：默认**展开**。
  - 子 section（full header）：默认**折叠**，点击 ▸ 按需展开。
  - 切换分类时所有 section 回到默认态（key 含 activeCategoryId 强制 remount）。
- **空文件夹**：会显示「空文件夹」提示，保留目录结构可见。

### 6.3 添加书签

两种入口：

1. 根 section 的书签网格末尾有一个 **+ 卡片**。
2. 子 section header hover 时右侧有 **+** 按钮。

依次输入：URL → 标题（默认值是 URL）→ 创建。

### 6.4 卡片操作

每张卡片：

- **整张点击**：在新标签页打开 URL。会自动记入「最近使用」。
- **拖拽排序**：按下 5px 阈值后开始拖拽（≤ 5px 视为单击），仅在同一分类内排序。
- **菜单（hover 右上角 ⋯）**：编辑、添加/编辑备注、删除。

#### 编辑模式

进入编辑模式后：

- **标题** 与 **URL** 两个字段可同时修改；都不能为空。
- 左上角的图标也可点击进入 IconPicker 改图标。
- **Enter 保存** / **Esc 取消**。
- 完全没有改动时直接退出，不写库。

#### 备注

- 已有备注：卡片底部展示备注（最多 2 行 line-clamp），点击编辑。
- 无备注：hover 卡片时底部出现 +「添加备注」按钮。
- 备注通过浏览器原生 prompt 输入；输入空字符串则清除备注。

#### 删除

确认弹窗里会显示书签标题，避免误删。

### 6.5 文件夹卡片

子文件夹会作为一种特殊卡片显示在「文件夹」区。点击进入子分类（与点击侧栏同步选中）。

### 6.6 全库搜索（暂未实装在主搜索框）

> 当前顶栏的搜索框仅用于网页搜索；未来 V0.2 会在搜索框右侧加切换按钮。
> 内部已实现 `searchKeyword` store 字段，会按**标题 + URL 模糊匹配**全库书签。

## 8. 数据持久化

### 8.1 存储位置

| 数据 | 存储 | 说明 |
| --- | --- | --- |
| 分类、书签、用户设置 | `chrome.storage.local`（key 前缀 `tabit:*`） | 跨浏览器进程持久化 |
| 最近使用记录 | `chrome.storage.local` | 同上 |
| 当前选中的搜索引擎 | `localStorage` | 仅当前域可读 |

### 8.2 容量上限

- `chrome.storage.local` 单浏览器扩展上限 **5MB**。
- 因此对图片资源做了限制：
  - **壁纸 2MB**（base64 后实际占用约 1.3 倍）
  - **图标 100KB**（图标用量大，需更严格）
- 大量书签 + 自定义图标的极端场景下，可能仍会超限；建议优先用 emoji 或图片 URL（不存内容只存链接）。

### 8.3 备份策略

强烈建议定期使用「**导出配置文件**」做本地备份，特别是在：

- 进行大规模整理前
- 切换设备前
- 升级扩展版本前

### 8.4 卸载扩展会发生什么

`chrome.storage.local` 的数据会随扩展卸载一并清除，且**不可恢复**。重新安装后是空白状态，需要重新导入备份或浏览器书签。

## 9. 故障排查

### 9.1 Firefox 卡在「加载中…」

历史问题，已修复（v0.1.0+）。如仍遇到：

- 确认使用 `pnpm build:firefox` 而不是直接用 chrome 包。
- 在 `about:debugging` 重新临时载入。
- 检查浏览器控制台是否有 `chrome is not defined` 错误，若有请提交 issue。

### 9.2 侧栏宽度拖完松手又跳回去

历史问题，已修复（v0.1.0+）。根因是首次 mount 时 `useEffect` 把默认值写覆盖了已存的真实值。如重现请提交 issue。

### 9.3 从浏览器导入后看不到分类

- 检查浏览器原生书签是否真的有内容（`chrome://bookmarks` / `about:bookmarks`）。
- 看看左侧是否在「批量管理」模式或「重命名」模式，这两种状态会改变交互。
- 强制刷新一下新标签页（`Ctrl+R` / `Cmd+R`）。

### 9.4 自定义图标 / 壁纸上传报「图片过大」

- 图标上限 100KB，壁纸上限 2MB。
- 解决：在线压缩工具（如 tinypng）压缩后再上传；或改用 URL 形式（不占本地容量）。

### 9.5 删除分类后想恢复

目前**没有撤销**功能（已在 ROADMAP 中规划）。强烈建议依赖「导出配置文件」做定期备份。

## 10. 隐私与安全

- Tab It 是**完全本地**的扩展，所有数据存在浏览器本地，**不会上传到任何服务器**。
- 「从浏览器导入书签」需要 `bookmarks` 权限，仅用于读取你的书签数据，不上报。
- 「favicon」在 Chrome 通过 `chrome://favicon` 协议获取（需 `favicon` 权限，仅 Chrome 申请；Firefox 走 Google s2 公共服务降级）。
- 网页搜索是直接 `window.open` 跳转到对应搜索引擎，**不经过 Tab It 中转**，搜索词不留任何记录。
